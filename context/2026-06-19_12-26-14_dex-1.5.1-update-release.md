# Dex 1.5.1 update release context

## User Intent

严格审查当前所有变更；无阻塞问题后提交并推送；重新打包发新版 Dex 1.5.1。

## Scope

- Include Dex fallback MSI update stall fix.
- Bump release version sources to `1.5.1`.
- Build and verify local Windows MSI.
- Push commit to `main`.
- Publish and verify GitHub Release assets for updater availability when credentials and workflow allow.

## Release Boundary

- Local MSI generation is not the same as updater-ready release publication.
- Updater-ready release requires tag `v1.5.1`, GitHub Release, signed `.sig`, and `latest.json` uploaded by `.github/workflows/release-assets.yml`.
- Do not run the local MSI installer on this machine during validation.

## Review Focus

- Old Dex Manager and launcher must not hold `%LocalAppData%\Programs\Dex` while MSI replaces files.
- Fallback MSI should queue a helper, exit Manager, and then launch `msiexec`.
- Version sources must agree: Cargo workspace, Cargo.lock local packages, manager `package.json`, `package-lock.json`, Tauri config, and web preview mock.
- Context artifacts should not contain secrets or raw private material.
